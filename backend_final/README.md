# Eventum Mobile — Backend Final

Объединённая версия бэкенда, включающая весь функционал основной версии (`backend/`) и интеграцию с 1С (`backend_eventum/`).

## Функционал

### Основной (из backend/)
- Регистрация / Логин (JWT)
- Профиль пользователя (обновление, интересы, аватар)
- Избранное, подписки на организаторов
- Лента событий (с модерацией)
- Создание / редактирование / удаление событий
- Просмотры событий (с защитой от ботов и rate-limiting)
- Посты, комментарии, голосования
- Покупка билетов, история билетов
- Чаты (личные сообщения через SocketIO)
- Уведомления (в реальном времени через SocketIO)
- Админ-панель (модерация событий/постов, бан пользователей, аналитика)
- Загрузка аватаров и изображений событий (с `PUBLIC_URL` для reverse proxy)

### Интеграция с 1С (из backend_eventum/)
- Поля синхронизации (`is_synced`) у User, Event, Ticket
- Двойное поле цены: `price` (актуальная) и `price_value` (совместимость)
- Фиксация `purchase_price` при покупке билета
- API-шлюз для 1С (`/api/v1/sync/*`) с проверкой по ключу
- Админские эндпоинты `/api/admin/sync-1c` и `/api/admin/manual-sync` с ODataService

## Запуск

### 1. Создание базы данных
```bash
# Отредактируйте ROOT_PASS в init_db.py (пароль суперпользователя PostgreSQL)
python init_db.py
```

### 2. Инициализация таблиц и создание администратора
```bash
python reset_db.py
```

### 3. Запуск в режиме разработки
```bash
python app.py
```

### 4. Запуск в продакшене
```bash
gunicorn --worker-class eventlet -w 1 app:app --bind 0.0.0.0:5001
```

## Переменные окружения (.env)

| Переменная | Описание |
|---|---|
| `DATABASE_URL` | Строка подключения к PostgreSQL |
| `JWT_SECRET_KEY` | Секретный ключ для JWT токенов |
| `CORS_ORIGINS` | Разрешённые источники CORS |
| `PUBLIC_URL` | Публичный URL сервера (для ссылок на загрузки за reverse proxy) |
| `ADMIN_PASSWORD` | Пароль администратора (используется в reset_db.py) |
| `SYNC_API_KEY` | Ключ для API-шлюза синхронизации с 1С |
| `ODATA_URL` | URL OData сервиса 1С |
| `ODATA_LOGIN` | Логин для 1С |
| `ODATA_PASS` | Пароль для 1С |

## Эндпоинты синхронизации с 1С

Все эндпоинты требуют заголовок `X-API-KEY` с корректным ключом.

| Метод | URL | Описание |
|---|---|---|
| GET | `/api/v1/sync/users` | Несинхронизированные пользователи |
| GET | `/api/v1/sync/events` | Несинхронизированные события |
| GET | `/api/v1/sync/tickets` | Несинхронизированные билеты |
| GET | `/api/v1/sync/all` | Все несинхронизированные данные |
| POST | `/api/v1/sync/confirm` | Подтверждение синхронизации (сброс `is_synced`) |
| POST | `/api/v1/sync/update_event_status` | Смена статуса модерации из 1С |

## Технологический стек

- **Flask** + **Flask-SocketIO** (eventlet)
- **SQLAlchemy** + **PostgreSQL**
- **JWT** (Flask-JWT-Extended)
- **ODataService** для интеграции с 1С
- **FireBase** для push-уведомлений (в приложении)
